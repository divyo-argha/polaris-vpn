VERSION 0.1 — MVP: SSH SOCKS5 TUNNEL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

WHAT IT DOES:
Wraps SSH dynamic port forwarding (-D flag) into a beautiful CLI experience.
Opens an encrypted SOCKS5 proxy on 127.0.0.1:1080 through any SSH server
the user provides. No server software needed — just SSH access to any VPS.

COMMANDS TO BUILD:

  spectre start --server user@host [--port 1080]
    1. Check and display current public IP (before)
    2. Spawn: ssh -D 1080 -N -o StrictHostKeyChecking=no
               -o ServerAliveInterval=30 -o ExitOnForwardFailure=yes user@host
    3. Wait for tunnel to be ready (retry connect to 127.0.0.1:1080)
    4. Re-check public IP through the proxy to confirm it changed
    5. Display: old IP → new IP, proxy address, uptime ticker
    6. Save PID to ~/.spectre/tunnel.pid
    7. Save config (server, port, start time) to ~/.spectre/config.json

  spectre stop
    Read PID from ~/.spectre/tunnel.pid, kill the process, clean up

  spectre status
    Show: tunnel up/down, current IP, server, uptime, proxy address

  spectre check
    Run a 3-point privacy report and print a pass/fail table:
    1. IP check      — fetch https://api64.ipify.org?format=json
    2. DNS leak      — resolve a known domain via Node dns.resolve,
                       compare against expected (flag if ISP DNS detected)
    3. IPv6 leak     — fetch https://ipv6.icanhazip.com with 3s timeout,
                       warn if an IPv6 address is returned
    Output: coloured table with ✓ PASS or ✗ FAIL per check

  spectre add <alias> --server user@host
    Save a named server profile to ~/.spectre/servers.json

  spectre list
    Show all saved server profiles

  spectre use <alias>
    Switch the active profile

VERSION 0.2 — TLS TUNNEL + DNS PROTECTION
(build ONLY after v0.1 is complete and tested)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

New features over v0.1:

1. TLS ENCRYPTED TUNNEL
   Replace SSH SOCKS5 with a proper TLS 1.3 encrypted tunnel.
   Ship a companion package spectre-server (separate npm package):
     npx spectre-server start   — runs on the VPS, generates self-signed
                                   TLS cert on first run, listens on :8443
   The spectre CLI connects to spectre-server over TLS and routes
   traffic through it. Fallback to SSH mode if TLS server not detected.

2. DNS-OVER-HTTPS (DoH)
   Intercept DNS queries locally using the dns2 package.
   Start a local DNS resolver on 127.0.0.1:5353.
   Forward all queries to Cloudflare 1.1.1.1 via HTTPS.
   User-configurable resolver (1.1.1.1, 8.8.8.8, or custom).
   This prevents ISP from seeing domain lookups even without WireGuard.

   New commands:
     spectre dns start    Start the local DoH resolver
     spectre dns stop     Stop it
     spectre dns status   Show resolver status and upstream

3. AUTO-RECONNECT
   Detect tunnel drop via keepalive pings every 30 seconds.
   Auto-reconnect with exponential backoff (max 5 retries).
   Emit a desktop notification (node-notifier) when reconnect happens.

4. ENHANCED LEAK CHECK
   Expand spectre check with:
   - WebRTC leak detection hint (explain in output, link to browserleaks.com)
   - Show which DNS server is being used (ISP vs DoH)
   - Display tunnel mode (SSH / TLS)

   New dependency: dns2, node-notifier

VERSION 0.5 — WIREGUARD FULL TUNNEL
(build ONLY after v0.2 is complete and tested)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

New features over v0.2:

1. WIREGUARD TUNNEL
   Full system-wide VPN using WireGuard — captures ALL OS traffic,
   not just proxied apps. Requires sudo/admin.

   spectre start --mode wireguard   Full system tunnel (needs sudo)
   spectre start --mode proxy       TLS proxy fallback (no sudo)
   spectre start                    Auto-detect best available mode

   Implementation:
   - Use wireguard-tools npm package for config generation
   - Generate keypairs locally, push public key to server automatically
   - On Linux: call wg and ip commands (via child_process)
   - On macOS: use wireguard-go userspace implementation
   - Create wg0 interface, set AllowedIPs = 0.0.0.0/0 for full tunnel
   - Persist WireGuard config to ~/.spectre/wg/wg0.conf

2. SERVER PROVISIONING — spectre deploy
   One command to fully configure a fresh Ubuntu VPS as a WireGuard server:

   spectre deploy --server ubuntu@<ip>

   This command must:
   a. SSH into the server (using ssh2 npm package)
   b. Update apt packages silently
   c. Install wireguard, ufw
   d. Enable IP forwarding (net.ipv4.ip_forward=1)
   e. Generate server keypair
   f. Write /etc/wireguard/wg0.conf with correct PostUp/PostDown iptables rules
   g. Enable and start wg-quick@wg0
   h. Configure UFW: allow 51820/udp, allow SSH, enable
   i. Generate a client config and save to ~/.spectre/wg/client.conf
   j. Print success with the server's public IP and WireGuard public key
   k. Automatically run spectre start --mode wireguard to connect

   Provider hints (shown in spectre deploy --help):
   - Oracle Cloud Free Tier (recommended, free forever)
   - Hetzner CAX11 (~€4/month, ARM, 20 Gbps)
   - DigitalOcean Droplet ($6/month)
   - Any Ubuntu 22.04 VPS with SSH access

3. MULTI-PEER SUPPORT
   spectre peer add <name>     Generate a new WireGuard peer config
   spectre peer list           Show all peers and their last handshake
   spectre peer remove <name>  Revoke a peer
   spectre peer qr <name>      Print QR code to terminal (qrcode-terminal)

   Peer configs are added to the server's wg0.conf via SSH automatically.
   Users can scan the QR code with the official WireGuard mobile app.

4. KILL SWITCH
   spectre config set kill-switch true

   When enabled, drops all non-tunnel internet traffic if WireGuard
   disconnects, preventing accidental IP exposure.
   Linux: iptables rules blocking non-wg0 traffic
   macOS: pf rules

   New dependencies: wireguard-tools, ssh2, qrcode-terminal

VERSION 1.0 — STEALTH MODE + POLISH
(build ONLY after v0.5 is complete and tested)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

New features over v0.5:

1. AMNEZIAWG STEALTH MODE
   Replace standard WireGuard with AmneziaWG when available on server.
   AmneziaWG obfuscates packet headers and adds junk traffic to defeat
   Deep Packet Inspection (DPI) — critical for users in countries with
   restrictive internet (Bangladesh, India, etc.).

   spectre deploy --stealth    Installs AmneziaWG instead of WireGuard
   spectre start --stealth     Connects using AmneziaWG protocol

   Stealth parameters to configure (randomised by default for best evasion):
   Jc (junk packet count), Jmin, Jmax (junk packet size range),
   S1, S2 (init packet sizes), H1-H4 (packet header values)

   Fallback: if server doesn't have AmneziaWG, fall back to WireGuard
   with a warning.

2. SERVER HEALTH DASHBOARD
   spectre status --full

   Shows in a table:
   - Tunnel mode (AmneziaWG / WireGuard / TLS / SSH)
   - Connected since + uptime
   - Current IP + server location (geoip lookup)
   - Bytes sent / received since connection
   - Ping latency to server
   - Kill switch status
   - DNS mode (DoH / system)
   - All peers and last handshake times

3. LOCAL REST API
   spectre server start   Starts a local REST API on 127.0.0.1:7070

   Endpoints:
   GET  /status     — tunnel state, IP, uptime
   POST /connect    — start tunnel
   POST /disconnect — stop tunnel
   GET  /peers      — list WireGuard peers

   Allows browser extensions, scripts, and other tools to control spectre.
   Document the API in README.

4. BANDWIDTH MONITOR
   spectre monitor   Live terminal dashboard showing:
   - Upload / download speed (bytes/sec) through the tunnel
   - Total data used this session
   - Estimated monthly usage at current rate
   - All peers activity (bytes per peer)

5. FIRST-CLASS WINDOWS SUPPORT
   Full WireGuard NT driver integration.
   Detect Windows and use wireguard.exe instead of wg CLI.
   PowerShell-friendly output for all commands.

6. AUTO-UPDATE
   spectre update   Check npm for newer version and prompt to install.
